1. 高层需求

本轮会话的高层目标，不是继续扩写 reviewer/operator 前端，而是沿着已经落地的 Runtime V2 主线，把真正的 runtime 自愈闭环补实。

主线要求分三层：

- 保持 Runtime V2 control plane 已落地能力稳定，不回头重开抽象重构。

- 把 Memory Governance 和 Chapter Workbench 视为辅助产品面，不再占用当前最核心的主线节奏。

一句话概括：

当前主线 = Runtime V2 已落地控制面 + 自愈 incident/repair 闭环 + 最小必要产品面
