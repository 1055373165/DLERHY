这轮已经实际落下并验证通过的增量是：

- runtime 现在已经能从 incident 自动生成结构化 `repair plan`
- 这份 repair plan 明确包含：
  - patch surface
  - owned files
  - validation command
  - bundle manifest / revision name / rollout scope
  - replay boundary / replay scope
- review deadlock 和 export misrouting 这两条已落地自愈链，现在都不再把修复动作散落在 controller 里，而是先生成统一 repair handoff
- patch proposal 和 incident 的 `status_detail_json` 里现在都会保留这份 repair plan，供后续真正的 repair agent / repair lane 消费
- 在此基础上，runtime 现在还会显式创建 `repair_dispatch`
  - proposal 一创建就会带 `pending` repair dispatch
  - 当前已落地的 review deadlock / export misrouting 自愈链会先走 `claim -> execute -> validate -> publish`
  - proposal / incident 的 `status_detail_json` 会同时保留 execution history、validation、bundle publication 结果
- 在此基础上，runtime 现在还会把 `repair_dispatch` 显式绑定到 claimable `REPAIR` work item
  - repair work item 当前使用 `stage=REPAIR` + `scope_type=ISSUE_ACTION`
  - `scope_id=proposal_id`，输入 bundle 中保留真正的 target scope / replay boundary / validation command
  - 当前已落地自愈链现在是通过现有 work-item 生命周期完成 `claim -> start -> validate -> publish -> replay -> succeed`
- 在这次推进后，repair 执行所有权已经从 controller 内部同步收口，移动到了 executor-owned `REPAIR` lane
  - review deadlock 和 export misrouting 现在都会先 schedule repair
  - 然后在提交后由 repair lane 领取并执行
  - repair work item 只有在 validate / publish / finalize 全部成功后才会落成 `SUCCEEDED`
- 在这次推进后，又进一步补上了显式 repair worker contract
  - executor 现在通过 `RuntimeRepairWorker` 运行 repair lane，而不是继续把完整 repair 流程硬塞在 `DocumentRunExecutor` 里
  - repair work item 输入 bundle 现在明确包含 `claim_mode / claim_target / dispatch_lane / worker_hint / worker_contract_version`
- 在这次推进后，又进一步补上了显式 repair worker registry
  - runtime 现在会按 `worker_hint / worker_contract_version` 解析 repair worker，而不是在 executor 里直接写死 `RuntimeRepairWorker`
  - unknown worker hint / unsupported contract version 会通过 repair work-item failure lifecycle 落成确定性的 terminal failure，不再绕过 lane 审计
- 这意味着当前已经从“只有 repair plan”推进到“runtime 自己持有 repair dispatch / execution lineage + 可领取 repair work-item surface + executor-owned repair lane + 明确 repair worker contract + 显式 repair worker registry”
