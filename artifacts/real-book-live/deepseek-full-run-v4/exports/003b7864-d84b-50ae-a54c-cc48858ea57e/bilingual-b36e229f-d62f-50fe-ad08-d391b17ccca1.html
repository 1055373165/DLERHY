1be9d403-6f77-57ca-ac85-88315409c57a1.2.1 What is an LLM?1.2.1 什么是大语言模型？

96572b6e-e679-5d86-828d-07805fd40f26The core principle of LLMs is simple: learn to predict the next word by processing massive amounts of text data.LLM的核心原理很简单：通过处理海量文本数据来学习预测下一个单词。
c6c7e82d-aa00-55eb-96cd-0b42b29550ffGiven the sentence "Because it's summer vacation, I didn't go to ______," most would expect the blank to be filled with "school." Through large-scale training on this simple principle, LLMs develop an understanding of language structure, context, and meaning.给定句子“因为是暑假，我没有去______”，大多数人会期望空白处填上“学校”。通过基于这一简单原理的大规模训练，LLM发展出对语言结构、上下文和意义的理解。
3f7a1dcd-b4c1-5921-85ba-6ef0be70cf8aLLMs process text by breaking it into tokens, which are typically words or word fragments.LLM通过将文本分解为标记（通常是单词或单词片段）来处理文本。
d110c6e5-d3de-5f87-90cc-425cc2e06970The model predicts the next token based on all preceding tokens.模型基于所有先前的标记来预测下一个标记。

613dac36-d3cf-5550-a6d0-2fa58c04950f1.5 Context engineering1.5 上下文工程

2b7e95b6-4b1d-5dc8-b925-93b5797295dcAn LLM predicts the next token based solely on the information within its context window.大型语言模型仅基于其上下文窗口内的信息来预测下一个标记。
13c314a8-77c4-5dc3-9348-2ff9a55277c0The model generates text by leveraging patterns learned during training to make use of the information in the context.该模型通过利用训练期间学习到的模式来利用上下文中的信息生成文本。
15b5b1b9-c4a8-5bb9-9426-00884b9d3f47This simple fact is why context engineering is so critical in agent development.正是这一简单事实，使得上下文工程在智能体开发中变得至关重要。

5a3687a7-f0d6-5879-a7b8-a10709c18d2e1.5.2 From prompt engineering to context engineering1.5.2 从提示工程到上下文工程
419d2896-37b8-5ca2-8bd6-89f9369829c4In the early days of LLM adoption, prompt engineering was the primary focus.在大型语言模型（LLM）应用的早期阶段，提示工程是主要关注点。
58d13c61-909d-502a-9e44-c7cd777f742cPractitioners concentrated on how to write system prompts that would make models respond more accurately and which instructions would yield outputs in the desired format.实践者专注于如何编写系统提示，以使模型响应更准确，以及哪些指令能产生符合期望格式的输出。
54ea80cf-8d2e-5d74-9d95-c4b60c219c06Techniques like "Let's think step by step" and "You are an expert" emerged during this period.在此期间，诸如“让我们一步步思考”和“你是一位专家”等技术应运而生。
9b06aafe-b616-5d01-af32-1c864f765dbcHowever, the landscape changed as agents began using tools.然而，随着智能体开始使用工具，这一格局发生了变化。
37d2ca53-d3f6-5fd1-b269-e8c7c3e426b4When an agent performs a web search, the results are added to the context.当智能体执行网络搜索时，搜索结果会被添加到上下文中。
17d82903-4c68-5b21-ab94-c8a2e7b5ac6eWhen it executes code, the execution results are added.当它执行代码时，执行结果会被添加进来。
b3f6ad4c-0aa2-54c1-a87d-3798e89c0ff0When it queries a database, the query results are added.当它查询数据库时，查询结果会被添加进来。
e74385d4-4b3e-57e1-a035-f85812b5cb2dContext began to change dynamically, evolving beyond static system prompts.上下文开始动态变化，超越了静态系统提示的范畴。
e7a73c90-57b5-5ac9-b4f0-09b4748c6328This shift raised a new question: "How do we maintain only the essential information needed for task completion in the context?" The approach to answering this question is context engineering.这一转变引发了一个新问题：“在上下文中，我们如何仅保留完成任务所必需的关键信息？”解决这一问题的途径就是上下文工程。
3c40c053-87b8-5ea9-9352-c43207081f20Figure 1.5 LLMs can only produce accurate, high-quality responses when sufficient information is provided in the context.图1.5：只有当上下文提供了足够信息时，大语言模型才能生成准确、高质量的回应。
03c13403-5b1e-5c2c-ba8b-dc3f0c63e12aContext engineering is the discipline of providing the information an LLM needs to perform its tasks at the right time and in the right form.情境工程是一门学科，旨在为大型语言模型提供执行任务所需的信息，确保这些信息在恰当的时间以合适的形式呈现。
be76daa0-e37f-5651-ad49-912758e53870It goes beyond simply writing good prompts to comprehensively designing how tool execution results should be processed, how much conversation history to retain, and when to retrieve external knowledge.它超越了仅仅编写优质提示的范畴，全面涵盖了如何设计工具执行结果的处理方式、保留多少对话历史，以及何时检索外部知识。
c1ba28b1-b5ab-5e3c-8159-fa695ff664541.5.3 Bigger context is not always better1.5.3 更大的上下文并不总是更好
2c212a41-0e9c-5c7e-9cc6-2efe0686c112The context windows of recent LLMs have grown dramatically.近期大型语言模型的上下文窗口已显著扩大。
258852c6-dcde-5cdb-8252-f3100df97434Most models now support context windows ranging from hundreds of thousands to a million tokens.如今大多数模型支持的上下文窗口范围从数十万到一百万个标记不等。
f6314440-3e27-5665-a401-68492e36a6dcSo would filling the entire context window with all available data yield the best responses?那么，将整个上下文窗口填满所有可用数据，是否就能得到最佳响应呢？
e7e51cd1-365c-508f-9171-52f70c6e0f4aUnfortunately, no.遗憾的是，没有。
ca805bca-68c8-512e-8fe5-bd08f602cef9Recent research has reported that model performance degrades as context length increases.近期研究表明，随着上下文长度增加，模型性能会下降。
f72a7245-5a68-5959-99a2-1ad49474cb25This phenomenon is sometimes called Context Rot.这种现象有时被称为“上下文腐化”。
41823cac-41f1-5358-a3ef-e17f0573c521The "Lost in the Middle" effect is particularly well documented, where models tend to miss important information located in the middle of long contexts.“迷失在中间”效应已得到充分记录，即模型容易忽略长上下文中位于中间部分的重要信息。

37ff8513-6515-5055-b3a0-ac0476a82af1This provides an important insight.这提供了一个重要的洞见。
78f73a2b-1d23-5c7b-b8ea-c123f0151bedTo maximize model performance, we should not simply provide more information but rather selectively provide only highly relevant information.为了最大化模型性能，我们不应简单地提供更多信息，而应选择性地仅提供高度相关的信息。
55d8f172-367f-5a64-afd9-455e41670f9aWhen the context is filled with unnecessary information, crucial details can get buried, or the model's attention can become dispersed.当上下文充斥着不必要的信息时，关键细节可能被埋没，或者模型的注意力可能变得分散。
d0503ec5-ac91-5d90-808b-bf99047da6e01.5.4 Five context engineering strategies1.5.4 五种上下文工程策略
6304c3fd-a01f-5e54-add9-ff793cf3e7bcContext engineering can be broadly categorized into five strategies.情境工程可大致分为五种策略。
dac2d07f-4241-58f1-88d5-1661085f4a13Each strategy is covered in detail across various chapters of this book.本书各章节将详细探讨每种策略。
3ecefa0b-7238-50e1-b3eb-89c813db8deeGeneration: Utilizing LLM-generated outputs within the context.生成：在上下文中利用LLM生成的输出。
e96da13e-9d8c-51c0-ad12-25ff88040813This includes generating plans for complex tasks or reflecting on completed work to revise strategies.这包括为复杂任务生成计划，或对已完成的工作进行反思以修订策略。
31b56a15-d118-5a53-9f39-7fd56b812534Beyond simply retrieving external information, this enables the LLM to structure and evolve the context on its own.这不仅限于简单地检索外部信息，还使LLM能够自行构建和演进上下文。
5a87a1a1-509c-5e32-afde-9f754f0afbfcRetrieval: Fetching necessary information from external sources and adding it to the context.检索：从外部来源获取必要信息，并将其添加到上下文中。
889ee6e2-39dc-56a1-bfff-2bceda9c393aThis includes web searches, database queries, file reading, and similar document retrieval from vector databases.这包括网络搜索、数据库查询、文件读取以及向量数据库中的类似文档检索。
0d15a5d9-f0ad-5795-9b38-5c1da58b434aThis is a core strategy that enables agents to access up-to-date information or domain-specific knowledge not present in their training data.这是一种核心策略，使智能体能够获取其训练数据中未包含的最新信息或领域特定知识。
7a289815-8ae9-5d0d-8b62-b86c968877e3Write: Recording information from the context to external storage.写入：将上下文中的信息记录到外部存储中。
1c3c33f4-a4e6-5e9f-83ff-e47133b3f39fThis includes saving important conversation content to long-term memory, recording intermediate calculation results to a scratchpad, or saving generated code to files.这包括将重要的对话内容保存到长期记忆中、将中间计算结果记录到草稿本中，或将生成的代码保存到文件中。
058739fe-b36a-5938-8c84-291cf802afbcThis overcomes the limitations of the context window and enables persistent storage of information.这克服了上下文窗口的限制，实现了信息的持久化存储。
07aa13c3-18b0-5e6d-b83f-20f07e3ccc79Reduce: Shrinking the size of the context.缩减：缩小上下文的大小。
6c6e0c0c-b814-5093-9598-ac077ffc2ff5This includes summarizing old conversations, deleting unnecessary information, or filtering information based on importance.这包括总结旧对话、删除不必要的信息，或根据重要性筛选信息。
35d81caa-291d-574c-8979-7b7209c970faThis is essential for preventing Context Rot and focusing the model's attention on key information.这对于防止上下文腐化并让模型聚焦于关键信息至关重要。
d75f7dee-2351-548e-8859-e9b4d472965aIsolate: Separating tasks or tools into distinct environments.隔离：将任务或工具分离到不同的环境中。
54f6d0f6-1389-55ed-ad3b-426ec9d67a1eThis includes executing complex code in sandbox environments or assigning different specialized domains to separate agents.这包括在沙盒环境中执行复杂代码，或将不同的专业领域分配给独立的智能体。
aef9e344-4e26-5d61-b3fa-1ccb027c8003This prevents any single context from becoming overly complex and enables the construction of optimized contexts for each task.这可以防止任何单个上下文变得过于复杂，并能为每项任务构建优化的上下文。
