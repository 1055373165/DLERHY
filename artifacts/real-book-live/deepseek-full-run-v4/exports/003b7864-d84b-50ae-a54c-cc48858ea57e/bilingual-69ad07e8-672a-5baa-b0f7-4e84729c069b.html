d572a8dc-3e8c-5a65-90b7-4a2d45dc3c81We can think of this note as a tool definition, a structured description that tells the LLM what tools (functions, APIs, or actions) are accessible, what parameters they expect, and what they do.我们可以将这个说明视为工具定义，即一个结构化的描述，告诉LLM哪些工具（函数、API或操作）是可访问的、它们需要什么参数以及它们的功能是什么。
d6d6e017-8612-5041-872b-513bab3019b7Once the LLM receives this information, it can begin reasoning about which tool is appropriate for a given task and how to use it, purely through text generation.一旦LLM接收到这些信息，它就可以开始推理对于给定任务哪个工具是合适的以及如何使用它，这一切都通过文本生成来完成。
